## About this part of the programme

This is **Part A — Python Foundations** of the AI Native Engineering programme.
It is the programming baseline for everything that follows. Total duration:
**30 hours across Weeks 1–8**.

Everything in this part is taught and practised in **Google Colab**, a free,
browser-based tool for writing and running Python. You do not need to install
anything on your laptop to follow along.

## Who this is for

- Freshers who know how to use a computer (browsing, typing, using apps) but
  have **no prior programming experience**.
- No prior knowledge of Python or any other programming language is assumed.

## What you need before you start

- A laptop/desktop with a working internet connection.
- A Google account (to use Google Colab and save your work to Google Drive).
- No software installation is required for this part.

## How this part is organised

Part A covers Modules I–V of the standard Python syllabus, plus Git and
professional tooling. Each module is broken into **Units**. Each Unit covers
one focused set of topics, with theory explained in plain language, followed
by hands-on practice.

| Module | Focus |
|---|---|
| P1 · Module I | Introduction — Python environment, variables, types, operators, output |
| P2 · Module II | Control Structures, Functions & Tooling — conditionals, loops, functions, functional constructs, packaging/tooling |
| P3 · Module III | Data Structures — lists, tuples, sets, dictionaries, iterators/generators/collections |
| P4 · Module IV | Classes & Objects — OOP foundations, inheritance & encapsulation, special methods & dataclasses |
| P5 · Module V | Files & Exception Handling — file I/O, errors & exceptions, robust file reader case study |
| P6 | Git, GitHub & Portfolio — version control basics, personal portfolio, Part A diagnostic |

## How to use this material

1. Read the Unit content in order — later Units build on earlier ones.
2. Open Google Colab alongside the material and type out every example
   yourself. Do not just read the code — run it.
3. If a term is new to you, it will be explained the first time it is used.
   You do not need to know anything beyond basic computer usage to start.
